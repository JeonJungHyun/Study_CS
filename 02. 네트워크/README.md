# 🌐 02. 네트워크

<details> <summary>1. 네트워크(Network)란?</summary> </br>
  
❶ 네크워크 </br>
: 두 대 이상의 컴퓨터(노드)들이 연결되어 데이터를 주고받을 수 있는 `디지털 통신망` </br>
단순히 선이 연결되어 있는 상태가 아니라, `정해진 규칙(프로토콜)`에 따라 데이터를 송수신할 수 있어야 한다. </br>

✏️ `"노드"` 란? </br>
네트워크에 속한 컴퓨터 또는 장비를 뜻하는 말로, 분산되어있는 컴퓨터를 통신망으로 연결한 것을 말한다. </br>
예를 들면, 개인 PC, 스마트폰, 프린터, 공유기 등이 있다. </br>
네트워크에서 여러 장치들은 노드 간 연결을 사용하여 서로에게 데이터를 교환한다. </br>
전세계가 연결되어있는 네트워크 중 하나가 인터넷으로, 인터넷은 네트워크의 한 종류이다. </br>

❷ 인터넷 vs 웹(www) </br>
- 인터넷 : 전 세계의 수많은 네트워크를 하나로 묶은 `가장 큰 네트워크` 그 자체. </br>
- 웹(www) : 인터넷이라는 도로 위에서 제공되는 `서비스` 중 하나. </br>
          HTTP 프로토콜을 기반으로 동작하며, 브라우저를 통해 접속하는 서비스 </br>

</details>

<details> <summary>2. 네트워크의 분류</summary> </br>
네트워크는 얼마나 넓은지(규모)와 어떻게 생겼는지(형태)에 따라 다음과 같이 분류된다. </br>

</br>

❶ 크기에 따른 분류 </br>
> 물리적인 거리와 연결 범위에 따라 나뉜다. </br>

- LAN(Local Area Network, 근거리 통신망) </br>
: 네트워크 매체를 이용하여 집, 사무실, 학교 등의 건물과 같은 가까운 지역을 한데 묶는 컴퓨터 네트워크 </br>
  - 장점: 데이터 전송 속도가 매우 빠르고, 에러 발생률이 낮다. </br>
  - 예시 : 집 와이파이, 같은 pc방에서 가까이 있는 친구와 함께 오버워치 게임을 하는 느낌. (멀리 떨어지면 통신이 멀어져서 불가능.) 

- WAN (Wide Area Network, 광역 통신망) </br>
: 드넓은 지리적 거리/장소를 넘나드는 통신 네트워크 또는 컴퓨터 네트워크 </br>
: WAN은 여러 개의 LAN을 통신할 수 있도록 연결시켜 놓은 것. </br>
  - 예시 : 네이버를 회사 또는 집에서 쓸 수 있음.

❷ 연결 형태에 따른 분류 </br>
> 장치들을 어떤 모양으로 배치했느냐에 따라 나뉜다. </br>

1. 스타형 (Star, 성형) : 중앙의 장비(공유기 등)에 모든 컴퓨터가 1:1로 연결된 형태. </br>
    - 장점 : 관리가 쉽고, 특정 컴퓨터가 고장 나도 다른 컴퓨터는 영향을 받지 않는다. </br>
    - 단점 : 중앙 장비가 고장 나면 **전체 네트워크가 마비**된다. </br>
    - 예시 : 가정집에서는 공유기를 통해 휴대폰, 노트북 등이 연결된다. </br>

2. 메시형 (Mesh, 망형) : 모든 노드가 서로 그물처럼 얽혀 연결된 형태. </br>

   🌟 실제 전 세계적인 인터넷 망의 핵심 구조이다. </br>
   
    - 장점 : 중앙에 있는 특정 장비와만 연결되는 것이 아니라 여러 노드가 연결되어 있기 때문에 </br>
    한 장비가 고장 났더라도 다른 우회로를 통해 데이터를 보낼 수 있어 신뢰성이 매우 높다. </br>

3. 버스형 (Bus) : 하나의 중앙 통신 회선(공통 버스)에 여러 노드가 붙어 있는 형태. </br>
    - 구조 : 하나의 중앙 통신 회선(공통 버스)에 여러 노드가 붙어 있는 형태. </br>
    - 특징 : 설치가 쉽고 비용이 저렴하지만, 회선 한 곳에 사고가 나면 전체가 영향을 받는다. </br>

4. 트리형 (Tree, 계층형) : 나무뿌리처럼 상위 노드에서 하위 노드로 가지를 치며 연결된 형태. </br>
    - 특징 : 확장이 쉽지만, 최상위(뿌리) 노드가 마비되면 그 하위 노드들은 통신이 불가능하다. </br>

5. 링형 (Ring) : 각각의 노드가 양 옆의 두 노드와 연결하여 전체적으로 고리와 같이 하나의 연속된 길을 통해 통신을 형태. </br>
    - 특징 : 모든 장치들이 토큰에 접근할 수 있으며 전송할 수 있는 기회를 갖게 되는 매우 순차적인 네트워크이다. </br>

<details> <summary>3. 네트워크의 통신 방식</summary> </br>
네트워크에서는 데이터를 `몇 명에게 보내는지에` 따라 통신 방식이 나뉜다. </br>

</br>

❶ 유니캐스트 </br>

✏️ 정의 : 하나의 송신자가 하나의 수신자에게 데이터를 보내는 방식. 즉,`1:1 통신` </br>
         가장 일반적이고 우리가 인터넷에서 사용하는 기본 방식이다. </br>

  1) 예시
    : 내가 네이버에 접속한다. 내가 유튜브 영상을 시청한다. </br>
  2) 단점 : 많은 사람에게 같은 데이터를 보내야 할 때 비효율적이다. </br>

❷ 브로드캐스트 </br>

✏️ 정의 : 하나의 송신자가 같은 네트워크에 속한 모든 장치에게 데이터를 보내는 방식. 즉, `1: 전체` </br>

  1) 예시 : 회사 사무실에서 “모두 들으세요!” 하고 외치는 상황.
  2) 예시2 : “192.168.0.5 누구야? 이걸 네트워크 전체에 물어본다. 그러면 해당하는 장비가 “나야!” 하고 응답한다. </br>
  3) 특징 : 같은 네트워크 내부에서만 동작하고 모든 장비가 데이터를 받아 네트워크 부하가 크다. </br>

❸ 멀티캐스트 </br>

✏️ 정의 : 하나의 송신자가 특정 그룹에 속한 장치들에게만 데이터를 보내는 방식. 즉, `1 : 특정 그룹` </br>
🤔 왜 필요할까? : 브로드캐스트는 너무 낭비이고 유니캐스트는 너무 많이 보내야 한다. -> “필요한 그룹에게만 보내자” </br>

  1) 예시 : 단체 카톡방에 메시지 보내는 상황. 화상 회의 시스템 </br>
  2) 특징 : 네트워크 효율적. 서버 부하 감소. 특정 그룹 제어 가능 </br>

<details> <summary>4. 프로토콜(Protocol)이란?</summary> </br>
프로토콜(Protocol)이란 네트워크에서 데이터를 주고받기 위한 일종의 약속, 양식이다. </br>

</br>

❶ 가까운 곳과 통신할 때 </br>
: Ethernet (MAC 주소) 

예) 같은 회사 내부 PC끼리. 같은 와이파이에 연결된 장치끼리

❷ 멀리 있는 곳과 통신할 때 </br>
LAN을 넘어 다른 네트워크와 통신할 때는 IP 기반 통신이 이루어진다. </br>

- IPv4 (IP 주소)
  : 목적지 주소를 지정. 어느 네트워크로 가야 하는지 결정. IP는 위치 정보다. </br>
   예) 192.168.0.10
- ARP
  : IP 주소를 → MAC 주소로 변환하는 역할. IP는 논리 주소. MAC은 실제 물리 주소 </br>
    같은 네트워크 안에서 최종 전달하려면 IP → MAC 변환이 필요함.
- ICMP
  : 에러 확인용 프로토콜. </br>
    예) ping google.com

🌟 정리 </br>
멀리 있는 곳과 통신할 때는 : </br>
IP 주소로 목적지 결정 -> ARP로 MAC 주소 변환 -> ICMP로 오류 확인 가능 </br>

즉,
> 멀리 있는 곳 통신 = IP 기반 통신 + 보조 프로토콜(ICMP, ARP)
