# 🧠 04. 알고리즘과 자료구조

<details> <summary>1. 알고리즘(Algorithm)이란?</summary> </br>
❶ 알고리즘이란? </br>
문제를 해결하기 위한 절차 또는 방법이다. </br>
즉, 입력을 받아 원하는 출력 결과를 만들어내는 일련의 과정이다. </br>
예를 들어 라면을 끓이는 과정도 하나의 알고리즘이다. </br>
물을 끓인다 -> 면과 스프를 넣는다 -> 일정 시간 후 완성 </br>
🌟 이렇게 단계적으로 문제를 해결하는 방식이 알고리즘이다. </br>
</br>

❷ 알고리즘의 조건 </br>
입력(Input) : 문제 해결을 위한 데이터 </br>
출력(Output) : 결과값 </br>
명확성 : 단계가 분명해야 한다 </br>
유한성 : 반드시 종료되어야 한다 </br>
효율성 : 가능한 빠르게 수행되어야 한다 </br>
</details>

<details> <summary>2. 자료구조(Data Structure)란?</summary> </br>
❶ 자료구조란? </br>
데이터를 효율적으로 저장하고 관리하기 위한 구조이다. </br>
알고리즘이 “어떻게 풀까”라면 자료구조는 “어떻게 담을까”이다. </br>
</br>
❷ 왜 중요한가? </br>
같은 데이터를 사용하더라도
어떤 자료구조를 쓰느냐에 따라 성능이 달라진다. </br>
예를 들어, </br>
배열 : 접근 빠름 </br>
연결 리스트 : 삽입/삭제 빠름 </br>
🌟 따라서 문제에 맞는 자료구조 선택이 중요하다. </br>
</details>

<details> <summary>3. 자료구조의 분류</summary> </br>
자료구조는 크게 두 가지로 나뉜다. </br>

</br>

| 구분       | 설명                 |
| -------- | ------------------ |
| 선형 자료구조  | 데이터가 일렬로 연결됨       |
| 비선형 자료구조 | 데이터가 계층 또는 네트워크 형태 |



</details>
